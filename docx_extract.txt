Objetivo del sistema
El sistema tiene como objetivo proporcionar al emprendimiento de suplementos deportivos una plataforma web para la gestión integral de su catálogo, pedidos y ventas, permitiendo que los clientes puedan explorar productos, generar pedidos y concretar la compra mediante contacto directo con el vendedor a través de WhatsApp.
La solución busca digitalizar y ordenar el proceso comercial del negocio, facilitando la toma de pedidos, el control de stock, la gestión de promociones y el seguimiento de ventas, sin depender inicialmente de una pasarela de pago automática.
Desde el punto de vista del cliente, el sistema permitirá consultar el catálogo de productos, filtrar y buscar suplementos, visualizar promociones vigentes, agregar productos a un carrito y generar un pedido, el cual será enviado automáticamente al vendedor mediante un mensaje prearmado de WhatsApp con el detalle completo de la orden.
Desde el punto de vista del emprendimiento, el sistema brindará un panel administrativo para gestionar productos, promociones, pedidos, clientes y stock, permitiendo registrar pagos, actualizar el estado de los pedidos, preparar envíos o retiros y mantener un control ordenado de las operaciones comerciales.
De esta manera, el sistema busca simplificar el proceso de venta para el negocio y mejorar la experiencia de compra del cliente, combinando la facilidad de uso de una tienda online con la flexibilidad de cerrar las ventas mediante comunicación directa con el vendedor.

A CONFIRMAR
Hace clic en “Finalizar pedido” y lo redirecciona a whatsapp con la orden de pedido con precios y observación en forma de mensaje.
El sistema:
guarda la orden en base de datos

Vendedor / admin
Recibe el mensaje con el detalle del pedido.
El stock queda reservado cuando el vendedor confirma disponibilidad y cambia el pedido a “Pendiente de pago”. El descuento definitivo del stock se realiza únicamente cuando el admin confirma el pago.
Confirma disponibilidad.
Pasa alias / CBU / link de pago.
El cliente paga.
El admin podrá actualizar el estado del pedido según la instancia comercial y de pago: pendiente de pago, pago informado, pago confirmado o rechazado/cancelado.
Al mismo tiempo podrá definir el estado del pedido en :
Pendiente de confirmación El vendedor todavía no respondió o no validó el stock.
Pendiente de pago El vendedor pasó alias.
Pago informado El cliente indica que realizó el pago y puede adjuntar o enviar el comprobante para su verificación.
Pago confirmado El admin verificó el pago, el stock se descuenta del inventario.
En preparación Se está armando el pedido.
Despachado / enviado Ya salió para entrega.
Entregado Pedido finalizado.
Cancelado El cliente podrá cancelar un pedido mientras no se encuentre en estado Pago confirmado o En preparación.El administrador podrá cancelar pedidos en cualquier estado previo a la entrega.
Vencido Si el cliente no informa el pago dentro de las 24 horas posteriores al estado Pendiente de pago, el pedido podrá pasar automáticamente a estado Vencido, liberando la reserva de stock.
Se prepara / entrega / envía o avisar al cliente para retirarlo.
En el caso de qué el cliente tenga que retirarlo de manera presencial, se contacta mediante whatsapp para avisarle qué el pedido está preparado.
